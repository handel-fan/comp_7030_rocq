\section*{Summary}


This Research Grant Proposal will focus on the implementation and proofs of 
novel, cutting-edge Dependent Intersection type programming structures in Rocq, 
improving how expressive type systems are implemented and verified.

This work brings a currently theoretical type system that Dr. Downen and 
Shriya Thakur are developing into a practical, usable implementation in Rocq, 
where we can reason about and prove useful properties.

Dependent Types allow programmers to express program invariants in their types.
One case is a list dependent type encoding its length - preventing references to 
empty lists as if they had elements.

Dependent Intersection Types (DITs) allow for types where
we have terms with type $A \cap B$, and $B$ depends on $A$.
This lets us say: “this value has property A, and property B defined using A.”

DITs are not available natively in Rocq. As a result, 
programmers must simulate them using indirect workarounds, requiring extra 
manual proof steps and encoding tricks.

The work to encode these relationships via DITs directly into Rocq will
allow developers to write simpler proofs with fewer errors by baking these 
relationships directly into the type system.

This work also has implications for reducing the size of the kernel - in proof assistant
terms, what can be "assumed" as a base, to solve proofs.

By introducing a single DIT primitive, we can encode
the functionality into what was once manual workarounds.
This leads to a smaller and simpler kernel, the trusted core of 
the system. In Rocq, this makes proofs and code easier to audit and reason 
about for correctness.


\section*{Planned Work}
The planned work is first to port new dependent type structures that Dr. Downen and Shriya Thakur are working on
to Rocq. This involves implementing the proposed types in the proof assistant Rocq.

Next, what is planned is to prove certain properties about these new types which make them desirable to be used.
Some of these are: 

\begin{enumerate}
  \item soundness - The implementation guarantees that if type checking succeeds, the execution will 
    not yield type related runtime errors.
  \item adequacy - The implementation guarantees that the intended meaning of a type matches the actual
    behavior of the system.
\end{enumerate}

Finally, a compiler written in Haskell is planned to allow programmers to write and execute
programs using the new type system.
This involves implementing a tokenizer, parser, and type checker.

A good estimate for work to be done is implementation in the 1st month, proofs in the 2nd and 3rd months,
and a compiler for evaluation as a stretch goal.

\section*{Expected Results}

The expected results from this research effort are an implementation of aforementioned dependent type structures,
as well as proofs of their soundness and adequacy.

Lastly, a compiler to evaluate it, is a stretch goal. This compiler would be able to type check and reason about
the newly proposed types, and evaluate/run code.
